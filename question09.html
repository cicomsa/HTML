Clean Code:Refractoring

Refactoring

Refactoring is a disciplined technique for restructuring an existing body of code, altering its internal structure without changing its external behavior. You refactor code to make it look clean and readable: you remove all unecessary code and comments, try to make your code as DRY and KISS as possible and making sure you don't have any pyramids of doom in terms of indentation.

DRY

Dry is an acronym for "Don't Repeat Yourself". Writing dry code makes your code better maintainable. You don't have to update code in two places. It is more readable and reusable - you don't need to define stuff more than once and you reuse already written logic. Also, it's less expensive as it saves you space and time and makes unit testing easier. For more thoughts, see codementor.io

KISS

KISS is an acronym for "Keep It Simple Stupid" The simpler your code is, the simpler it will be to maintain it in the future, and of course, if other people see it, they will thank you for that. More thoughts? Have a look at simpleprogrammer.com or itexico.com

Separation of Concerns

In computer science, separation of concerns (SoC) is a design principle for separating a computer program into distinct sections, such that each section addresses a separate concern. In this case, your html files should only be organizing your content, whereas CSS should only be used for styling.
